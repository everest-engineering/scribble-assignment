# Data Model: Room Setup and Lobby

## Entities

### Room

Represents a single game session. Created when a player clicks "Create Room"
and destroyed when all participants have left.

| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `code` | `string` | Unique room identifier | 4-char alphanumeric (A-Z, 2-9, excluding 0/O/1/I). Auto-generated. |
| `status` | `"lobby" \| "active"` | Current room lifecycle state | Initial: `"lobby"`. Transitions to `"active"` via start game. |
| `hostId` | `string` | Participant ID of the room creator | Set at creation. Immutable. |
| `participants` | `Participant[]` | Players currently in the room | Max 8. Order is join sequence. |
| `createdAt` | `string` (ISO 8601) | When the room was created | Set once at creation. |
| `updatedAt` | `string` (ISO 8601) | Last modification timestamp | Updated on join, start, leave. |

**State transitions:**

```
lobby ──[host clicks start, 2+ players]──▶ active
lobby ──[last player leaves]──▶ (removed)
```

### Participant

A player who has joined a room. Each connection is a distinct participant.

| Field | Type | Description | Constraints |
|-------|------|-------------|-------------|
| `id` | `string` (UUID v4) | Unique participant identifier | Auto-generated by backend. |
| `name` | `string` | Display name in the lobby | 1-16 alphanumeric characters. Trimmed. |
| `joinedAt` | `string` (ISO 8601) | When the participant joined | Set once at join time. |

### RoomSnapshot

Public-facing view of a room sent to clients. Omits internal state.

| Field | Type | Description |
|-------|------|-------------|
| `code` | `string` | Room code |
| `status` | `"lobby" \| "active"` | Current room state |
| `participants` | `Participant[]` | Current participants |
| `hostId` | `string` | ID of the host participant |
| `availableWords` | `string[]` | List of possible secret words (for drawer visibility in future) |
| `roles` | `string[]` | Available roles (for drawer/guesser assignment in future) |

## Validation Rules

| Field | Rule | Source |
|-------|------|--------|
| Participant name | 1-16 alphanumeric chars, trimmed | Spec Q4 |
| Room code | Required, non-empty, must match active room | Spec FR-003, FR-004 |
| Participants per room | Max 8 | Spec FR-012 |
| Concurrent rooms | Max 100 | Spec FR-012 |
| Game start precondition | At least 2 participants, must be host | Spec FR-009, FR-010 |
