# Data Model: Room Setup & Lobby

**Feature**: `001-room-setup-lobby`
**Date**: 2026-05-31
**Source file**: `backend/src/models/game.ts`

---

## Entities

### Room

The primary aggregate. Stored in the in-memory `Map<string, Room>` in
`backend/src/services/roomStore.ts`.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `code` | `string` | 4-char, uppercase alphanumeric (A-Z excluding I/O, 2-9) | Generated by `generateUniqueCode()`; unique across all live rooms |
| `status` | `"lobby" \| "in-progress"` | Required | Initial value: `"lobby"` |
| `hostId` | `string` | Required, UUID | Set to the first participant's `id` at creation; **NEW field** |
| `participants` | `Participant[]` | Min 1 | Ordered by join time; first entry is always the host |
| `createdAt` | `string` | ISO 8601 | Set at creation, never mutated |
| `updatedAt` | `string` | ISO 8601 | Updated on every mutation |

**State transitions**:
```
lobby ──(POST /rooms/:code/start, host, ≥2 players)──▶ in-progress
```
No transition back. Once `in-progress`, the room accepts no new participants.

---

### Participant

Embedded in `Room.participants`. Not stored independently.

| Field | Type | Constraints | Notes |
|-------|------|-------------|-------|
| `id` | `string` | UUID (randomUUID) | Opaque identifier; returned to the joining client as `participantId` |
| `name` | `string` | Min 1 char after trim | Provided by the player at create/join time |
| `joinedAt` | `string` | ISO 8601 | Set at join time, never mutated |

**Host identification**: `participant.id === room.hostId` — no separate field
on `Participant` needed.

---

### RoomSnapshot (API-facing projection)

Read-only view returned by all API endpoints. Derived from `Room` at request
time. Adds `hostId` to the existing shape.

| Field | Type | Notes |
|-------|------|-------|
| `code` | `string` | |
| `status` | `"lobby" \| "in-progress"` | |
| `hostId` | `string` | **NEW field** — UUID of the host participant |
| `participants` | `Participant[]` | Same shape as stored |
| `availableWords` | `string[]` | From seed data; unchanged |
| `roles` | `ParticipantRole[]` | From seed data; unchanged |

## Validation Rules

| Rule | Location | Detail |
|------|----------|--------|
| `playerName` non-empty | Backend `schemas.ts` + frontend form | `z.string().min(1)`; frontend checks `trim()` before submit |
| `roomCode` non-empty | Frontend form | Checked via `roomCode.trim()` before submit |
| Room must exist | Backend `joinRoom`, `getRoom`, `startGame` | Returns 404 if code not found |
| Room must be `"lobby"` to join | Backend `joinRoom` | Returns 409 if `status === "in-progress"` |
| Caller must be host to start | Backend `startGame` | Compares `body.participantId` to `room.hostId`; returns 403 if mismatch |
| ≥2 participants to start | Backend `startGame` | Returns 400 if `room.participants.length < 2` |
