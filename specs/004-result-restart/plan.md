# Implementation Plan: Result, Restart, and Final Validation

**Branch**: `feat/scribble-spec-kit` | **Date**: 2026-05-31 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/004-result-restart/spec.md`

## Summary

When the room status transitions to `"ended"` (set by Scenario 3's correct-guess logic), the
game screen replaces its active-game content with a full result view: secret word prominently
displayed, per-participant scoreboard, and complete ordered guess history. The host additionally
sees a Restart button. When the host triggers a restart, a new `POST /rooms/:code/restart`
endpoint clears all round state (`drawerId`, `secretWord`, `guesses`, `scores`) and returns
the room to `"lobby"` status ÔÇö participants are preserved. All participants detect the status
change via the existing polling loop and automatically navigate back to `/lobby`.

## Phase 0: Research

All unknowns resolved. See [research.md](./research.md).

Key decisions:
- New endpoint: `POST /rooms/:code/restart` ÔÇö mirrors `POST /rooms/:code/start` pattern
- `restartRoom(code, participantId)` ÔÇö pure service function, throws `HttpError`
- Host check: `participantId !== room.hostId` Ôćĺ `HttpError(403, "Only the host can restart")`
- Status check: `room.status !== "ended"` Ôćĺ `HttpError(409, "Room is not ended")`
- Reset fields: `status Ôćĺ "lobby"`, `drawerId Ôćĺ ""`, `secretWord Ôćĺ ""`, `guesses Ôćĺ []`, `scores Ôćĺ {}`
- Participants preserved: `room.participants` unchanged by restart
- Result view: same `GamePage` URL, no new route; when `status === "ended"`, replace active-game content with result view
- Lobby navigation: `GamePage` polling effect detects `room.status === "lobby"` Ôćĺ `navigate("/lobby", { replace: true })`
- Restart button: only rendered when `isEnded && participantId === room.hostId`

## Phase 1: Design & Contracts

### Backend Changes

#### 1. `backend/src/api/schemas.ts`

Add:
```typescript
export const restartRoomSchema = z.object({
  participantId: z.string().uuid(),
});
```

#### 2. `backend/src/services/roomStore.ts`

New exported function `restartRoom()`:
```typescript
export function restartRoom(code: string, participantId: string): Room {
  const room = rooms.get(code);
  if (!room) throw new HttpError(404, "Room not found");
  if (room.status !== "ended") throw new HttpError(409, "Room is not ended");
  if (participantId !== room.hostId) throw new HttpError(403, "Only the host can restart");

  room.status = "lobby";
  room.drawerId = "";
  room.secretWord = "";
  room.guesses = [];
  room.scores = {};
  room.updatedAt = now();
  rooms.set(room.code, room);
  return cloneRoom(room);
}
```

#### 3. `backend/src/api/rooms.ts`

Add new route (after the existing `/guess` route):
```typescript
router.post("/:code/restart", (request, response, next) => {
  try {
    const { code } = roomCodeParamsSchema.parse(request.params);
    const { participantId } = restartRoomSchema.parse(request.body);
    const room = restartRoom(code.toUpperCase(), participantId);
    response.json({ room: toRoomSnapshot(room, participantId) });
  } catch (error) {
    next(error);
  }
});
```

### Frontend Changes

#### 4. `frontend/src/services/api.ts`

Add `restartRoom()` method to the api object:
```typescript
restartRoom(code: string, participantId: string) {
  return request<{ room: RoomSnapshot }>(`/rooms/${encodeURIComponent(code)}/restart`, {
    method: "POST",
    body: JSON.stringify({ participantId }),
  });
}
```

#### 5. `frontend/src/state/roomStore.ts`

Add `restartRoom()` method to `RoomStore`:
```typescript
async restartRoom() {
  if (!this.state.room || !this.state.participantId) return null;
  const response = await this.withLoading(() =>
    api.restartRoom(this.state.room!.code, this.state.participantId!)
  );
  this.setRoomSnapshot(response.room);
  return response.room;
}
```

#### 6. `frontend/src/pages/GamePage.tsx`

Three changes:

**a) Lobby navigation via polling**: Add to the navigation `useEffect`:
```typescript
useEffect(() => {
  if (!room) {
    navigate("/", { replace: true });
  } else if (room.status === "lobby") {
    navigate("/lobby", { replace: true });
  }
}, [navigate, room]);
```

**b) Full result view when `isEnded`**: Replace the current "Round Ended" banner with a
dedicated result section. When `isEnded`, render:
- Secret word displayed prominently
- Scoreboard (already wired to `room.scores`)
- ResultPanel / full guess history (already wired to `room.guesses`)
- Restart button ÔÇö only when `participantId === room.hostId`

**c) Hide active-game content when ended**: Existing guards (`!isEnded` for `GuessForm`)
are already in place; canvas area also shows placeholder for non-drawers. Ensure the full
layout is replaced or the result content is prominently positioned above the sidebar content.

## Risk Register

| Risk | Mitigation |
|---|---|
| Navigation `useEffect` fires when room is fetched on GamePage mount if room status is already "lobby" | Guard with `replace: true` ÔÇö harmless navigation to lobby; occurs only on very unlikely race condition |
| `room.scores = {}` on restart ÔÇö `toRoomSnapshot` spreads `scores` directly, so lobby snapshot returns `{}` which is valid | No change needed; lobby snapshot with empty scores is correct per spec FR-004 |
| `restartRoom` host check uses `participantId !== room.hostId` ÔÇö must match UUID string equality | Zod schema validates UUID format; hostId set at `createRoom` time as UUID string |
| Host clicks Restart while navigating away ÔÇö double navigation | `navigate("/lobby", { replace: true })` is idempotent; React Router handles gracefully |
| `toRoomSnapshot` for "lobby" status after restart: `isActive` and `isEnded` are both false ÔÇö snapshot omits `secretWord` and `wordPlaceholder` | Confirmed correct: lobby snapshot should not leak secret word |
