# Implementation Plan: Gameplay Interaction

**Branch**: `assignment-Anusha` | **Date**: 2026-05-31 | **Spec**: [spec.md](./spec.md)

**Input**: Feature specification from `/specs/003-gameplay-interaction/spec.md`

## Summary

Implements the active gameplay loop for a Scribble round: an HTML5 canvas that only
the drawer can interact with (locally — no broadcast this sprint), a validated guess
submission endpoint, and a polling-based guess history view that keeps all players in
sync. Scores are tracked in-memory per round, initialized at 0, and incremented by 100
for each correct guess.

## Technical Context

**Language/Version**: TypeScript 5.x — existing backend (Node.js + Express) and
frontend (React 18 + Vite)

**Primary Dependencies**:
- Backend: Express, Zod (validation already in use), uuid (already in use)
- Frontend: React, native HTML5 Canvas API (no new library needed — drawing is
  local-only this sprint, so a heavy canvas library would be overengineering)

**Storage**: Existing in-memory `Map<string, Room>` in `backend/src/services/roomStore.ts`.
Guesses and per-round scores will be stored as new fields on `CurrentRound`.

**Testing**: Manual browser testing per constitution. Each user story has an
independent test scenario documented in the spec.

**Target Platform**: Local development server — backend port 3001, Vite dev server for
the frontend.

**Project Type**: Web application — existing `backend/` + `frontend/` structure

**Performance Goals**: Guess history poll interval ≤ 3 seconds (per SC-003)

**Constraints**:
- REST over HTTP only; no WebSockets or SSE
- No external database; all state lives in the in-memory Room store
- No new npm packages unless no standard alternative exists
- Existing API contracts must not break (Brownfield Awareness)

**Scale/Scope**: Small local multiplayer room (handful of players)
