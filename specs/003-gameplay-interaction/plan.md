# Implementation Plan: Gameplay Interaction

**Branch**: `feat/scribble-spec-kit` | **Date**: 2026-05-31 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/003-gameplay-interaction/spec.md`

## Summary

When the game is active, guessers submit text guesses via a new `POST /rooms/:code/guess`
endpoint. Each guess is trimmed, validated, and compared case-insensitively against the secret
word. A correct guess awards 100 points to the guesser and transitions the room to `"ended"`;
an incorrect guess appends to history with 0 points. All guesses and scores are included in
every snapshot poll response so all participants see live updates. The drawer sees a local-only
drawing canvas (no server transmission). When the game ends, a "Round Ended" banner appears
and the secret word is revealed to all.

## Technical Context

**Language/Version**: TypeScript 5.x (Node.js 20 backend, Vite/React 18 frontend)
**Primary Dependencies**: Express 4, Zod, Vitest, React, `useSyncExternalStore`
**Storage**: In-memory `Map<string, Room>` — no persistence
**Testing**: Vitest (backend + frontend)
**Target Platform**: Local dev server (backend :3001, frontend :5173)
**Project Type**: Web service (REST) + SPA frontend
**Performance Goals**: Snapshot response < 50ms local; polling cadence ~2s
**Constraints**: No WebSockets, no DB, no auth. All new code fully typed (no `any`).
**Scale/Scope**: Single in-memory process, handful of concurrent rooms

## Phase 0: Research

All unknowns resolved. See [research.md](./research.md).

Key decisions:
- New endpoint: `POST /rooms/:code/guess` — mirrors `POST /rooms/:code/start` pattern
- `submitGuess(code, participantId, text)` — pure service function, throws `HttpError`
- Comparison: `trimmed.toLowerCase() === room.secretWord.toLowerCase()`
- Score storage: `Record<string, number>` initialized to `0` for all participants at game start
- Guess storage: `Guess[]` ordered array, `participantName` denormalized
- When `"ended"`: `secretWord` exposed to all viewers; `wordPlaceholder` omitted
- Canvas: new `DrawingCanvas` component, pointer events, local-only, no store writes

New exported function `submitGuess()`:
```typescript
export function submitGuess(code: string, participantId: string, text: string): Room {
  const room = rooms.get(code);
  if (!room) throw new HttpError(404, "Room not found");
  if (room.status !== "active") throw new HttpError(409, "Game is not active");
  if (!room.participants.some(p => p.id === participantId))
    throw new HttpError(403, "Participant not in room");
  if (participantId === room.drawerId) throw new HttpError(403, "Drawer cannot guess");

  const trimmed = text.trim();
  if (!trimmed) throw new HttpError(400, "Guess cannot be empty");

  const correct = trimmed.toLowerCase() === room.secretWord.toLowerCase();
  const guess: Guess = {
    participantId,
    participantName: room.participants.find(p => p.id === participantId)!.name,
    text: trimmed,
    correct,
    index: room.guesses.length,
  };

  room.guesses.push(guess);
  if (correct) {
    room.scores[participantId] = (room.scores[participantId] ?? 0) + 100;
    room.status = "ended";
  }
  room.updatedAt = now();
  rooms.set(room.code, room);
  return cloneRoom(room);
}
```

Update `toRoomSnapshot()` to include `guesses`, `scores`, and handle `"ended"` status:
```typescript
export function toRoomSnapshot(room: Room, viewerParticipantId?: string): RoomSnapshot {
  const isActive = room.status === "active";
  const isEnded = room.status === "ended";
  const isDrawer = isActive && viewerParticipantId === room.drawerId;

  return {
    code: room.code,
    hostId: room.hostId,
    status: room.status,
    participants: room.participants.map(p => ({ ...p })),
    availableWords: isActive && !isDrawer ? [] : listWords(),
    roles: [...STARTER_ROLES],
    drawerId: room.drawerId,
    guesses: room.guesses.map(g => ({ ...g })),
    scores: { ...room.scores },
    ...(isActive && isDrawer  ? { secretWord: room.secretWord } : {}),
    ...(isActive && !isDrawer ? { wordPlaceholder: [...room.secretWord].map(() => "_").join(" ") } : {}),
    ...(isEnded               ? { secretWord: room.secretWord } : {}),
  };
}
```

#### 4. `backend/src/api/rooms.ts`

Add new route (after the existing `/start` route):
```typescript
router.post("/:code/guess", (request, response, next) => {
  try {
    const { code } = roomCodeParamsSchema.parse(request.params);
    const { participantId, text } = submitGuessSchema.parse(request.body);
    const room = submitGuess(code.toUpperCase(), participantId, text);

    response.json({
      room: toRoomSnapshot(room, participantId)
    });
  } catch (error) {
    next(error);
  }
});
```

## Risk Register

| Risk | Mitigation |
|---|---|
| `toRoomSnapshot` spread order — `isEnded` secretWord branch must not conflict with `isActive` branches | Use mutually exclusive conditions: `isActive && isDrawer`, `isActive && !isDrawer`, `isEnded` |
| `scores[participantId]` may not exist if participant joined after `startRoom` (impossible per join guard) | Initialize `scores` via `Object.fromEntries(participants.map(...))` at start time |
| `Guess[]` and `Record<string, number>` not initialized in `createRoom` — TypeScript will flag | Add `guesses: []` and `scores: {}` to `createRoom` Room literal |
| Canvas `pointerup` event may not fire if pointer leaves the canvas boundary | Handle `onPointerLeave` and `onPointerUp` both to end stroke |
| `GuessForm` currently has no error display — needs local or store-sourced error | Use store `error` field (already provided by `withLoading`); reset on next submit attempt |
| Frontend `RoomSnapshot` type missing `guesses`/`scores` — TypeScript build will fail | Mirror both fields in `api.ts` before any component uses them |
